Lists

Everything in Python is an object

x = 3    # x references the object 3
y = 3    # y references the same object
z = y    # z references the same object
y = 4    # y now references the object 4

User defined classes and objects

Each time we create an object from a class, Python creates a new object in memory (Python decides where it is stored):

c1 = Card(3, 3)
c2 = Card(3, 3)    # Different object

Python Lists are array-based lists

A list in Python is a sequence of references to objects. The references in the list are maintained in a contiguous physical block in memory, packed one after another in sequence - called an array-based list.

Each index in the list references objects stored elsewhere such as 4 or “4”

Tuple

A tuple is a sequence of references to objects. A Tuple is immutable. Implementation is similar to Lists, but is able to take advantage of being immutable.

Space &Time Complexity

The amount of space required for the list depends on the number of elements, not the space required for the basic objects.

Searching for an element

Finding an element in a list takes O(n), where n is the length of the list

Increasing list size

If there is space at the end of the list, use it.
Appending items to a list may require reservation of new area in memory for the entire list, and time to copy the list elements across.

For a list that may grow to size n, naïve append take O(n2).

Policy: Each time we must grow the list, double its size.
Doubling the size each time space is needed takes O(n) to build a list of size n.
Average cost of a single append is then O(1).

Popping an element

For a list of length n, 
    pop(i) takes (n-1-i) copies.
    pop(0) takes n-1 copies
    pop is O(n)
